## 1.1 Personal Background and Application Process～个人背景与申请历程
>[!NOTE]
>我本科前两年半比较摆烂。家里希望读研，但我没法保研（成绩一般而且两门挂科，没有竞赛/项目），又不想备考考研。  
>所以大三用半年补项目和比赛，同时考雅思；大四全年去实习，也因此进入了大模型方向。
- **升学途径**：
	- 保研：拿下保研名额-> 各种高校夏令营+找相关老师联系-> 保研升学（直博）~ 科研
	- 考研：定目标院校和专业 -> 笔试 -> 面试 -> offer ~ 科研
	- 留学：提交资料申请 -> 等待/笔试+面试/面试-> offer -> 留位费 -> 学费 ->  签证+入学 ~ 科研/上课

- **Master / MPhil 区别**：
	- Master：授课型，上课，课程为主，官网申请
	- MPhil：研究型，跟老师做项目，更适合读 PhD，需要提前联系导师，门槛更高需要一些connection，但对成绩要求可能还好，主要是看科研能力

- **申请专业方向**：计算机 / 人工智能 / 数据科学

- **学校定位**：主申港校～港前五
	- 后期也想过美国、新加坡、以及 MPhil
	- 但我转大模型方向比较晚，申请时间不够，需要 gap + 联系导师，最后就不考虑

- **GPA**：**3.46/5.00**
	- 不算高，所以靠项目、竞赛、实习这些软背景来补
- **雅思**：**6.5**（听力7.5、阅读7.5、写作6.0、口语5.5）

- 简历核心就是：==**GPA + 雅思 + 软背景**==（项目/竞赛/科研/实习）  
	- 个人申请体感：master 申请里 GPA 和雅思（以及学校）最关键，软背景是拉分项；港大我也没上，估计门槛没过去

## 1.2 个人建议

- 先想清楚你读研的目标，越早越不焦虑：  
   - 水个专业混个学位好找工作, 不一定对口；
   - 找对口专业对口就业；
   - 准备读 PhD 但不想国内学制；
   - 或者就是想出国体验
- 申请定位：  
	- GPA 和雅思够的话可以冲高级专业/高级院校；
	- 不够就用项目/竞赛/实习补软背景，广撒网，一些交叉融合专业也可以考虑  
	- 选港/新/澳/英/美主要看预算和需求（生活、认可度、未来打算）
- 雅思/托福尽量早点刷出来：  
  不同项目要求不一样，早点刷出来省心；英语提升多靠长期接触（论文、博客、论坛）
- 联系好老师写推荐信
- 预算够又想省事，可以先找中介：  
  先帮你理清脉络和定位；DIY 需要时间成本学习沉淀一下基本流程，不过有AI在问题很简单了  
  我前期找过指南者，需要的话可以找我来推荐一下负责人，在广州；后期我就自己改了自己投递
- 小红书有很多经验贴，可以当参考


# 2. 未来实习就业（计算机相关）

>[!NOTE]
>- 先确定你是想读 PhD 还是想找工作。  
>- 我在科研和实习两边都做过，目前也是在两者并行在做。学术和工业差距真的很大。学术研究往往不用考虑实际应用的问题，主要就是做实验，写故事；工作的话要考虑成本收益人力各种问题，需要工程化的业务思维。
>- 我比较喜欢把科研当兴趣去做，不在乎能否产出论文，主要是学习一些前沿的路线以及有实际意义（可应用）的探索（例如我毕设做的是在小模型上的deepseek-r1-zero训练方法复现和对比实验；我和同事构建了一套短剧剪辑+配音的强化学习训练以及llm as judge的框架；训练dino做了识集的模型框架推理）；
>- 总体而言，我更喜欢做应用工程的事情，相对来说更有正反馈，但无论走哪条路，消除信息差非常重要

- 想做科研：尽早定方向、跟老师做项目、发论文，往 PhD 路径走  
	- 非热门方向 PhD 多数会走科研或高校教职，现在学术界也很卷了
	- 热门方向phd基本上都是大模型相关，基本都是各方面拉满的选手，慎入

- 想找工作：尽早确定岗位方向开始学习，刷实习做项目准备面试
	- 算法：大模型（基座训练 / 应用 / infra）、推荐、数据挖掘 、数据分析
	- 开发：后端（Java/Go）、客户端、数据开发、前端  
	- 测开，运维，etc
	- 产品/运营（技术类，产品类）
